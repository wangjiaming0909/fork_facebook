350  // Upstream API

352  // The interfaces defined below update the virtual stream based priority
353  // scheme from the current system which allows only strict priorities to a
354  // flexible system allowing an arbitrary tree of virtual streams, subject only
355  // to the limitations in the HTTP/2 specification. An arbitrary prioritization
356  // scheme can be implemented by constructing virtual streams corresponding to
357  // the desired prioritization and attaching new streams as dependencies of the
358  // appropriate virtual stream.
359  //
360  // The user must define a map from an opaque integer priority level to an
361  // HTTP/2 priority corresponding to the virtual stream. This map is
362  // implemented by the user in a class that extends
363  // HTTPUpstreamSession::PriorityMapFactory. A shared pointer to this class is
364  // passed into the constructor of HTTPUpstreamSession. This method will send
365  // the virtual streams and return a unique pointer to a class that extends
366  // HTTPUpstreamSession::PriorityAdapter. This class implements the map between
367  // the user defined priority level and the HTTP/2 priority level.
368  //
369  // When the session is started, the createVirtualStreams method of
370  // PriorityMapFactory is called by HTTPUpstreamSession::startNow. The returned
371  // pointer to the PriorityAdapter map is cached in HTTPUpstreamSession. The
372  // HTTP/2 priority that should be used for a new stream dependent on a virtual
373  // stream corresponding to a given priority level is then retrieved by calling
374  // the HTTPUpstreamSession::getHTTPPriority(uint8_t level) method.
375  //
376  // The prior strict priority system has been left in place for now, but if
377  // both maxLevels and PriorityMapFactory are passed into the
378  // HTTPUpstreamSession constructor, the maxLevels parameter will be ignored.
379 
380  // Implments a map from generic priority level to HTTP/2 priority.
381  class PriorityAdapter {
382  public:
383  virtual folly::Optional<const HTTPMessage::HTTPPriority>
384  getHTTPPriority(uint8_t level) = 0;
385  virtual ~PriorityAdapter() = default;
386  };
